### Separation of Concerns
- **Incoming**: Macros notify dashboard
- **Outgoing**: Dashboard controls via webhook
- **Tracking**: Macros log your actions

- Knows all phone numbers
- Has conversation context
- Controls routing
- Tracks everything

### MacroDroid is the Hands
- Receives SMS
- Sends SMS
- Detects events
- Executes commands

### Result
- âœ… No routing errors
- âœ… No stale variables
- âœ… Full control
- âœ… Complete audit trail

## ğŸ” Troubleshooting Guide

### Customer not receiving AI response?
```
Check:
1. Macro 1 triggered? (Check logs)
2. Dashboard received POST? (Check terminal)
3. Dashboard called webhook? (Check logs)
4. Macro 2 triggered? (Check logs)
5. SMS sent? (Check phone)
```

### Dashboard send not working?
```
Check:
1. MACRODROID_WEBHOOK_URL in .env.local?
2. Dashboard restarted after env change?
3. Webhook URL correct?
4. Macro 2 enabled?
5. Test webhook with curl
```

### AI not pausing after manual reply?
```
Check:
1. Macro 3 enabled?
2. Macro 3 triggered? (Check logs)
3. Dashboard received POST? (Check terminal)
4. Message shows as "staff" in dashboard?
5. Conversation status = "manual"?
```

### Missed call response not sending?
```
Check:
1. Macro 4 enabled?
2. Macro 4 triggered? (Check logs)
3. API returned response? (Check terminal)
4. Variable saved correctly?
5. SMS action executed?
```
